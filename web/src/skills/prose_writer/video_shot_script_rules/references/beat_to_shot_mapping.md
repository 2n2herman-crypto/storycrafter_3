# 场景节拍 → 镜头 输入处理说明

本规则指导镜头设计时，以引擎拼接好的序列细纲为输入，按细纲内的场景分段处理：

1. 定位细纲里对应的场景小节，取其时空边界、场景目标、冲突与障碍作为本场景镜头设计的宏观背景。
2. 取该场景小节下已嵌入的全部节拍块。
3. 将场景背景 + 该场景的节拍串一并作为上下文，逐节拍推进拆解镜头——具体每拍拆几个镜头、怎么拆，结合 `shot_split_rules` 的分镜拆分原则、`visual_description_rules` 的视觉转化铁律自主判断，不预设固定拆分比例。
4. 时长估算按 `duration_estimation` 的公式表逐镜头计算。

注意：本文件不包含任何转译判断本身——判断逻辑完全由 LLM 结合其余三份规则性 reference 在推理中完成。
