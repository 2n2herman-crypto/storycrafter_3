# v6.4-1 前端交互改造方案

## 一、改造目标

写作期（Phase Gate lock 后）的资产栏应以「剧本正文章节」为主产出物，同时保持「大纲设计资产」的可选可见性。核心需求：

1. **资产栏折叠切换**：以「大纲设计」和「剧本正文」两大组为单位可折叠/展开
2. **写作期默认视图**：锁定后资产栏自动折叠设计组、展开正文组
3. **章节卡片列表**：`chapters/*.md` 按序列分组展示，支持章节序号、状态、字数等元信息
4. **锁定角标**：被冻结的设计资产在卡片上显示 🔒 锁定标识
5. **写作期只读提示**：用户点击设计资产卡片时展示「当前只读」提示而非可编辑状态

---

## 二、现状分析

### 已有基础（v6.3）

- `AssetCardPanel` 已支持 `CollapsibleSection` 折叠组件（`COLLAPSIBLE_GROUPS = new Set(['大纲切片', '剧本正文'])`）
- `assetStore.getAssetList()` 已对 `chapters/*` 分到「剧本正文」组、`sequences/*` 分到「大纲切片」组
- `phaseStore` 提供 `isWriting()` / `isLockedPath()` / `getBaseline()` 接口
- `useAssetStore.selectedPath` + `onSelect` 驱动 CurrentPanel 切换

### 缺口分析

| 缺口 | 说明 |
|------|------|
| 折叠状态无持久化 | 当前 `CollapsibleSection.expanded` 是组件本地 useState，切换卡片/刷新页面丢失 |
| 写作期默认折叠设计组 | 无 phase → expanded 映射逻辑 |
| 章节卡片缺乏元信息 | `AssetCardData` 只有 `path/filename/group/status`，无字数/序列归属/锁定状态 |
| 锁定角标无视觉表现 | 卡片无 locked/unlocked 视觉区分 |
| 只读设计资产点击行为 | 点设计资产卡仍进 CurrentPanel 展示内容（ok），但无"只读"提示 |
| 无"全部章节"概览 | 写作期缺少一个"已写 X/Y 章"的总体进度 |

## 三、改造方案

### 3.1 折叠状态迁移：`uiStore` + phase 联动

在 `uiStore` 增加折叠状态，并让 `useEffect` 监听 phase 变化自动切换：

```ts
// uiStore 增量
interface UiState {
  collapsedSections: Record<string, boolean> // key=groupName
  toggleSection: (group: string) => void
  setSectionCollapsed: (group: string, collapsed: boolean) => void
}
```

**联动规则**：
- `designing → writing` 锁定时 → `setSectionCollapsed('大纲设计', true)` + `setSectionCollapsed('剧本正文', false)`
- `writing → designing` 解锁时 → 两组都置为 `false`（展开），方便回到设计期操作
- 用户在 writing 期手动展开设计组 ⇢ 不受限（UI 展示 read-only 快照）

### 3.2 资产组分类调整

当前分组名：

| 组名 | 包含 |
|------|------|
| `大纲切片` | `sequences/*.md` |
| `剧本正文` | `chapters/*.md` |
| `角色设定`/`世界观设定`/... | 各静态资产 |
| `（空）` | 未匹配路径 |

**v6.4 合并分组逻辑**：

设计期（phase=designing）：保持现有分组粒度（角色设定/世界观设定/幕/序列/伏笔/支线 + 大纲切片 + 剧本正文各自独立折叠），默认全部展开。

写作期（phase=writing）：增加「大纲设计」父级折叠。所有非「剧本正文」的组（静态设计资产 + 大纲切片）收入「大纲设计」父折叠下作为子项。用户一键折叠/展开所有设计资产，内部子组仍可独立操作。

```
写作期资产栏结构：
├── 📁 大纲设计 [折叠/展开]     ← 父级折叠，默认折叠
│   ├── 角色设定                ← 子组独立折叠，默认折叠
│   ├── 世界观设定              ← 同上
│   ├── 幕结构
│   ├── 序列清单
│   ├── 伏笔追踪
│   ├── 支线管理
│   └── 大纲切片
└── 📁 剧本正文 [展开]          ← 父级折叠，默认展开
    ├── chapters/S1-1.md
    ├── chapters/S1-2.md
    └── ...
```

实现方式：`CollapsibleSection` 支持 `children` 嵌套渲染。写作期用 `ParentCollapsibleSection` 包装非正文组（复用 CollapsibleSection 的折叠 UI，props 增加 `defaultCollapsed`）。

父级折叠默认折叠（phase=writing），子级折叠保持独立状态。

### 3.3 章节卡片增强

`AssetCardData` 类型添加可选字段（不破坏现有接口）：

```ts
// types/index.ts 增量
export interface AssetCardData {
  path: string
  filename: string
  group: string
  status: AssetStatus
  // v6.4 可选扩展
  locked?: boolean        // 是否被 phase gate 冻结
  wordCount?: number      // 正文字数（仅 chapters/*）
  metaInfo?: string       // 额外展示文本（如 "S1-1 所属"）
}
```

**字数统计**：`assetStore.refreshFile` 时若路径是 `chapters/*`，计算 `${content}` 的中文字数（`/[一-鿿]/g` 正则）存入 `wordCount` 字段。非中文/非章节文件不计算。

**锁定标识**：`getAssetList()` 调用 `phaseStore.isLockedPath(path)` 设置 `locked`。

### 3.4 锁定角标与只读提示

#### 角标视觉

`AssetCard` 组件内，当 `data.locked === true` 时在卡片右下角显示 🔒 小锁 icon：

```tsx
// AssetCard.tsx 增量
{data.locked && <span className={styles.lockBadge}>🔒</span>}
```

样式：`.lockBadge` 右侧定位、字号 0.7em、半透明。

#### 只读提示

设计期行为不变。写作期用户点击设计资产卡片：

1. `selectCard(path)` 正常读取内容并设置 `selectedPath`
2. `CurrentPanel` 顶部显示蓝色信息条：

```
🔒 当前为设计期资产的锁定快照。如需修改，请点 HeaderBar 解锁回设计期。
```

信息条仅在设计资产被选中且处于 writing 期时出现，用 `phaseStore.isLockedPath(selectedPath)` 判断。

### 3.5 写作期默认布局转换

`App.tsx`（或 `MultiColumnLayout`）监听 `phaseStore.isWriting() / isDesigning()`：

- `designing → writing`：
  - 若当前 `selectedPath` 是设计资产 → 自动选中第一个 `chapters/*` 卡片（若某章节存在），否则置空
  - 刷新资产列表确保 `chapters/*` 可见
  - 重置折叠状态为"大纲设计折叠、剧本正文展开"

- `writing → designing`：
  - selectedPath 不变（保留当前查看的章节）
  - 折叠状态全展开

### 3.6 写作期进度概览

HeaderBar 右区已显示「已写 X 章 / 共 Y 序列」文案。v6.4 增强：

- 点击进度文案展开下拉小面板（tooltip 风格），列出每个序列的章节完成状态：

```
S1-1 ✅ 2,340字  S1-2 ✅ 3,100字
S1-3 🔄 生成中  S2-1 ⬜ 未开始
...
```

- 数据来源：`act_map.md` 解析序列总数（现状已有），`fm.listAssetFiles()` 过滤 `chapters/*` 逐个读入算字数
- 无需新 store，在 `HeaderBar` 组件内本地计算 + useState 缓存

## 四、边界情况

| 场景 | 行为 |
|------|------|
| writing 期无任何 chapters/ 文件 | 剧本正文组折叠且灰色显示「暂无章节，开始写作吧」；用户点开折叠组后显示空状态占位 |
| 写作期解锁回设计期后 chapters/ 保留 | 折叠状态全展开；剧本正文组仍可见但无锁定角标 |
| writing 期用户手动点了折叠所有组 | 尊重用户操作，不强制展开正文组（非首次锁定时的 cover 行为只在 lock() 瞬间触发一次） |
| 字数统计 0 或读取失败 | 显示 `--` 而非 0 |
| 序列尚未生成场景切片 | 该序列在所有进度展示处显示 ⬜ 未开始 |
