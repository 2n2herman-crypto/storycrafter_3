# 世界观一致性检查

你是质检 subagent 当前通过 `read_skill` 读取到的检查规则之一。你的职责是检查故事内容是否与世界观设定一致。

## 检查维度

### 1. 角色设定 vs 世界观
- 角色的身份、能力、知识范围是否不违反世界的核心规则（如世界观设定"魔法已消失三百年"，不能出现一个独立魔法师角色未经解释）
- 角色所处的社会地位是否与其行为逻辑一致（如设定里规定"贵族才可骑马"，平民角色不能随意出现骑马场景）

### 2. 剧情 vs 世界观
- 剧情中的关键转折是否依赖了违反世界观设定的便利性（如设定科技水平为"蒸汽时代"，出现电子通讯即为违规）
- 冲突逻辑是否被世界观中的物理/社会规则正确约束

### 3. 场景细节 vs 世界观
- 场景描述中的环境要素是否与 `worldbuilding.md` 一致（时令、地理、文明特征等）
- 同一地点在不同序列中是否有矛盾描述（如上个序列说"城市被围墙包围"，下个序列出现"从南门出城后看到一望无际的麦田"——那围墙呢？）

## 输出格式

```
# 世界观一致性检查报告

## 1. 角色设定 vs 世界观
- [scenes/S2-2.md] 角色"赵明"在场景中使用了无线通讯设备，但 worldbuilding.md 设定技术水平为"1980年代固定电话" → 建议改为有线电话或飞鸽传信

## 2. 剧情 vs 世界观
- [sequences/S1-1.md, sequences/S1-2.md] worldbuilding.md 设定"每年仅仲夏一个月可出海"，但 S1-1 场景为初春（时空边界）却有出海场景 → 建议调整时间设定或补充特殊解释

## 3. 场景细节 vs 世界观
- [scenes/S1-1.md → scenes/S3-1.md] 同一城市 S1-1 描述为"被高墙包围"，S3-1 描述"从窗台能看到连绵山脉"——若无明确说明围墙内存在高地，两者矛盾 → 建议统一

## 汇总
- 设定违规：N
- 剧情违规：M
- 细节矛盾：K
```

每个发现附带：问题定位 + 世界观设定的原文引用 + 违规描述 + 建议修复方向。
