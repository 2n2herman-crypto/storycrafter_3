# v6.4-4 开发实施步骤

## 实施策略

v6.4 四条工作线**按顺序执行**，后一步依赖前一步的基础：

```
Wave A (Engine 扩展)     →   Wave B (SKILL.md 改写)
  ↓                               ↓
Wave C (前端交互改造)     ←   Wave D (写作期默认布局联动)
```

- **Wave A** 和 **Wave B** 可并行推进（A 改 engine，B 改 prompt，互不依赖）
- **Wave C** 依赖 A（相邻序列标签需要 engine resolveExtraContext 就绪才能测试）
- **Wave D** 依赖 B + C（UI + phase 联动需要前两波完成）

建议实施顺序：**A → B → C → D**（稳妥串行），或 **(A+B) → C → D**（A/B 并行后合并入 C）。

3. 新增方法 `extractBehaviorTrack(target: string, output: string)`：

```ts
private extractBehaviorTrack(target: string, output: string): void {
  const regex = /<!--\s*BEHAVIOR_TRACK:\s*(.+?)\s*-->/g
  const tracks: string[] = []
  let match: RegExpExecArray | null
  while ((match = regex.exec(output)) !== null) {
    tracks.push(match[1].trim())
  }
  if (tracks.length > 0) {
    this.behaviorTrack.delete(target)  // 先删再插，保证 LRU 插入顺序正确（Map.set 不改变已有 key 的位置）
    this.behaviorTrack.set(target, tracks)
    // 最多保留 5 章追踪记录
    while (this.behaviorTrack.size > 5) {
      const oldest = this.behaviorTrack.keys().next().value
      if (oldest) this.behaviorTrack.delete(oldest)
    }
  }
}
```

### A-3 视听转化软校验

**文件**：`src/orchestrator/orchestratorEngine.ts` + `src/types/index.ts`

**改动内容**：

1. 在 `OrchestratorEngine` 类新增方法 `runSoftValidation(output: string): string[]`：

```ts
private runSoftValidation(output: string): string[] {
  const warnings: string[] = []

  // 1. 全知叙述句式
  if (/他不知道[，,\s]*这是[他她].*?[最后终]/.test(output)) {
    warnings.push('检测到疑似全知叙述句式（如"他不知道，这是他最后一次..."）')
  }

  // 2. 直白情感标签
  const emotionMatches = output.match(/她?很(?:伤心|难过|生气|愤怒|害怕|紧张|开心|高兴|失望|焦虑|恐惧)/g)
  if (emotionMatches && emotionMatches.length > 0) {
    warnings.push(`检测到 ${emotionMatches.length} 处直白情感标签：${emotionMatches.slice(0, 3).join('、')}。建议用行为替代`)
  }

  // 3. 过长无对话描写
  const paragraphs = output.split(/\n\n+/)
  for (let i = 0; i < paragraphs.length; i++) {
    const lines = paragraphs[i].split('\n').filter(l => l.trim())
    if (lines.length > 5 && !/[「「"」"」：]/.test(paragraphs[i])) {
      warnings.push(`检测到第 ${i + 1} 段超过 5 行的无对话描写段落`)
      break
    }
  }

  return warnings
}
```

2. 在 `script_writer` 执行成功后调用：

```ts
// 在 tool complete 后，result.success 为 true 时（extractBehaviorTrack 调用后）

if (softWarnings.length > 0) {
  // 将 warnings 追加到已有的 event.warnings（由 executeTool 回调透传）
  // event 已有 warnings 字段则合并，否则新建
}
```

3. `src/types/index.ts` 中 `ExecutionEvent` 新增字段：

```ts

  // ... 现有字段
  warnings?: string[]  // v6.4 软校验警告，非阻塞
}
```

**验收**：故意让 LLM 产出含"他不知道，这是他最后一次..."或"她很伤心"的正文，确认 execution log 中出现软校验警告。

### A-4 边界处理

| 场景 | 行为 |
|------|------|
| 同幕只有 1 个序列 | `<same_act_sequences>` 只含 1 个 `<slice>`，Writer 正常使用 |
| target 是首章（无前章） | `<previous_chapter_draft>` 为空（标签被 appendExtraLabels 自动过滤） |
| 本章无 BEHAVIOR_TRACK 注释 | behaviorTrack 不记录本章条目 |
| behaviorTrack 超 5 章后 rewrite 第 1 章 | `delete(target) → set(target, …)` 保证该条目移到最后，不被误淘汰 |
| LLM 未输出任何软校验违禁模式 | `runSoftValidation` 返回空数组，无 warning 写入 |
| reset_all 触发 | behaviorTrack.clear() |
