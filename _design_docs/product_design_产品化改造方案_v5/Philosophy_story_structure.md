# 幕 → 序列 → 场景 → 节拍 层级结构梳理

---

## 一、幕（Act）的定义信息
需要输入{story_structure.md}，并混合其中多种叙事的功能框架，生成包括3-12幕的幕结构。
| 信息维度 | 定义内容 | 如何向下支持序列 |
|---------|---------|----------------|
| **幕编号/名称** | 第一幕/第二幕/第三幕（或更多） | 为序列提供宏观叙事方向 |
| **幕定位** | 故事宏观阶段的划分，由情节点(Plot Point)界定边界 | 序列必须落在对应幕的情感区间内 |
| **核心任务** | 第一幕：建置(Setup)；第二幕：对抗(Confrontation)；第三幕：解决(Resolution) | 序列的短期目标必须服务于幕的核心任务 |
| **情感弧线** | 第一幕：建立共鸣→打破平衡；第二幕：挣扎→成长；第三幕：决战→释放 | 序列的情感走向必须匹配幕的整体弧线 |
| **篇幅占比** | 通常1:2:1（第一幕25%，第二幕50%，第三幕25%） | 序列的篇幅分配受幕的总篇幅约束 |
| **转折点** | 幕与幕之间由重大转折点连接（情节点1、情节点2、中点等） | 序列的终点往往是幕内转折点或幕间转折点 |
| **主角状态** | 第一幕：日常状态→被抛出；第二幕：适应新世界→面对反派压力；第三幕：最终决战→回归/蜕变 | 序列中的主角行动必须推动主角在幕层面的状态迁移 |
| **世界状态** | 第一幕：平衡世界；第二幕：失衡世界；第三幕：新平衡世界 | 序列中的世界变化必须反映幕层面的世界状态变迁 |

### 幕 → 序列 的支持逻辑（5条）

1. **宏观方向**：序列的短期目标必须服务于幕的核心任务（建置/对抗/解决）
2. **情感区间**：序列的情感走向必须匹配幕的整体弧线（平衡→打破→新平衡）
3. **篇幅约束**：序列的篇幅分配受幕的总篇幅约束（如第二幕占50%）
4. **状态迁移**：序列中的主角行动必须推动主角在幕层面的状态变化
5. **世界映射**：序列中的世界变化必须反映幕层面的世界状态变迁

## 二、序列（Sequence）的定义信息

| 信息维度 | 定义内容 | 如何向下支持场景 |
|---------|---------|----------------|
| **序列编号/名称** | 按幕内顺序编号，建议命名（如"绑架序列"、"监狱适应序列"） | 为场景提供中观叙事方向 |
| **序列定位** | 幕的下级单位，由多个场景组成，围绕一个统一目标或事件推进 | 场景必须落在序列的统一语境内 |
| **核心任务** | 完成一个短期目标（Sub-goal），作为通向终极目标的步骤 | 场景的微观目标必须服务于序列的短期目标 |
| **统一语境(Context)** | 序列只讲"一件事"：一个目标、一个时间段、一个地点、一个事件 | 场景的内容必须与序列的"一件事"直接相关 |
| **戏剧问题** | 用是非问句定义："主角能否达成X？"，序列结束时必须回答 | 场景中的冲突和转折必须逐步回答序列的戏剧问题 |
| **场景集合** | 包含2-N个场景，所有场景都服务于该序列的统一语境 | 场景的边界由序列的起承转合节奏决定 |
| **三幕微结构** | 每个序列内部也有起承转合：建置→上升动作→小高潮→回落 | 场景的结局必须推动序列向小高潮发展，并在序列终点提供Fresh News |
| **新鲜信息(Fresh News)** | 序列结束时必须提供新的信息，推动主角进入下一个短期目标 | 场景的篇幅分配受序列总篇幅约束 |
| **篇幅** | 通常8-15分钟（电影）或若干章节（小说） | 场景的情感基调必须匹配序列在微结构中的位置 |

### 序列 → 场景 的支持逻辑（7条）

1. **统一语境**：场景的内容必须与序列的"一件事"直接相关
2. **目标对齐**：场景的微观目标必须服务于序列的短期目标
3. **戏剧问答**：场景中的冲突和转折必须逐步回答序列的戏剧问题
4. **起承转合**：场景的边界由序列的微结构节奏决定（建置/上升/高潮/回落）
5. **Fresh News**：场景的结局必须推动序列向小高潮发展
6. **篇幅约束**：场景的篇幅分配受序列总篇幅约束
7. **情感匹配**：场景的情感基调必须匹配序列在微结构中的位置

## 三、场景（Scene）的定义信息

| 信息维度 | 定义内容 | 如何定义边界 |
|---------|---------|------------|
| **场景编号/名称** | 按序列内顺序编号，建议命名 | 编号决定其在序列中的位置 |
| **场景定位** | 序列的下级单位，是最基本的叙事积木单元 | 必须判断该场景是否服务于当前序列的统一语境 |
| **核心任务** | 推动故事前进，完成一个具体的微观叙事动作 | 如果场景不推动故事，则不属于该序列或应删除 |
| **场景目标(Objective)** | 角色进入场景时想要什么（必须明确） | 场景开始=角色带着目标进入；场景结束=目标结果确定 |
| **冲突与障碍** | 什么阻碍了角色达成目标 | 冲突是场景的动力源，无冲突=静态场景=边界模糊 |
| **场景结果(Outcome)** | 目标达成/被拒绝/改变，产生新的情境 | 结果是场景的出口条件，决定了下一个场景的入口状态 |
| **时空边界** | 通常单一地点+连续时间；地点/时间/人物群变化=新场景 | 时空变化是硬边界：换地点/跳时间/换人物群=新场景 |
| **节拍(Beats)** | 场景内部的情绪/关系/权力变化节点，是最小叙事单位 | 节拍是场景内部的软边界，标记情绪转变点 |
| **情感转变** | 场景开始和结束之间，角色的情感状态必须发生变化 | 情感无变化=场景可能冗余，边界需要重新界定 |

### 场景边界判断逻辑（7条）

1. **目标入口**：场景开始 = 角色带着明确目标进入
2. **冲突驱动**：冲突是场景的动力源，无冲突=静态场景=边界模糊
3. **结果出口**：结果是场景的出口条件，决定下一个场景的入口状态
4. **硬边界**：时空变化（地点/时间/人物群）= 新场景
5. **软边界**：节拍标记场景内部的情绪/权力/关系转变点
6. **冗余判断**：情感无变化 = 场景可能冗余，边界需重新界定
7. **张力曲线**：节拍的累积效应决定场景的情感结果

## 四、节拍（Beat）的定义信息

| 信息维度 | 定义内容 | 边界判断 |
|---------|---------|---------|
| **节拍定位** | 场景内部的最小叙事单位，是动作与反应之间的空间 | 节拍无独立时空边界，存在于场景内部 |
| **核心任务** | 标记场景内情绪、权力、关系或认知的关键转变 | 当角色的反应方式发生明显变化时，即产生新节拍 |
| **触发机制** | 一个动作（台词/行为）引发另一个角色的反应，形成节拍 | A说X→B以Y方式回应（节拍1）；A再说Z→B以W方式回应（节拍2） |
| **情感/关系转变** | 每次节拍都伴随微观的情感或权力位移 | 情感方向改变=新节拍；情感持续=同一节拍延续 |
| **叙事功能** | 节拍链构成场景的内在张力曲线 | 节拍的累积效应决定场景的情感结果 |

### 节拍内部逻辑（5条）

1. **动作-反应空间**：A说X → B以Y方式回应（节拍1）
2. **位移判断**：情感方向改变 = 新节拍；情感持续 = 同一节拍延续
3. **无独立边界**：节拍存在于场景内部，不跨越场景边界
4. **累积效应**：节拍链的张力累积决定场景的最终情感结果
5. **微观位移**：每次节拍都伴随情感/权力/关系的微小但可感知的位移

## 五、关键设计原则总结

### 幕层
- 用**"情节点"**做硬边界
- 用**"核心任务+情感弧线"**做方向约束
- 向下输出：**宏观方向 + 情感区间 + 篇幅约束**

### 序列层
- 用**"统一语境"**做内容筛选器
- 用**"戏剧问题"**做终点校验
- 用**"三幕微结构"**做节奏分配
- 向下输出：**统一语境 + 短期目标 + 起承转合节奏 + Fresh News要求**

### 场景层
- 用**"目标-冲突-结果"**做动力结构
- 用**"时空变化"**做硬边界
- 用**"情感转变"**做冗余判断
- 向下输出：**明确的入口/出口状态 + 张力需求**

### 节拍层
- 用**"动作-反应"**做触发机制
- 用**"情感/权力位移"**做边界判断
- 无独立时空边界，是场景内部的张力累积单元
