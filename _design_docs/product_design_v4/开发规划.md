### App.tsx 依赖注入变更

**v3 旧代码**：
```typescript
const fm = new InMemoryFileManager(DEFAULT_ASSET_PATHS)
const llm = new LLMClient()
const purifier = new IntentPurifier(llm)
const dispatcher = new AgentDispatcher(llm, fm)
initAssetStore(fm)
initChatStore(dispatcher, purifier, fm)
```

**v4 新代码**：
```typescript
const fm = new InMemoryFileManager(DEFAULT_ASSET_PATHS)
const llm = new LLMClient()
const engine = new OrchestratorEngine(llm, fm)
initAssetStore(fm)
initChatStore(engine)
```

## Step 6：打磨

> **目标**：LLM 端到端联调通过

### 测试清单

| 测试项 | 操作 | 预期结果 |
|--------|------|---------|
| FC 调用 | 输入"写个太空歌剧" | Orchestrator 调用了 generate_worldbuilding |
| TAG 校验 | Tool 返回含 TAG 的内容 | 正确提取内容，写入资产文件 |
| 状态流转 | 文件生成后 | 卡片状态变为 generated |
| modified 检测 | 再次写入同一文件 | 卡片状态变为 modified |
| 依赖链提示 | refine_worldbuilding 后 | 回复中提示下游可能需更新 |
| reset_all | "推翻重来" | 文件清空，卡片回到 pending |
| 复杂需求 | "完善世界观，生成角色和幕结构" | 5 轮内完成多 Tool 调用 |
