# 伏笔回收检查

你是质检 subagent 当前通过 `read_skill` 读取到的检查规则之一。你的职责是检查 foreshadowing.md 中的规划是否在序列/场景/节拍中被正确执行。

## 检查维度

### 1. 伏笔铺设完整性
比照 `foreshadowing.md` 的每条伏笔规划，检查：
- 每条标记"plant"的伏笔，在其声明的铺设序列中是否有对应场景/节拍实际承载了该信息披露
- 铺设方式是否合理（不是简单的一句话带过，需要有实际的叙事承载）

### 2. 伏笔回收完整性
- 每条标记"payoff"的伏笔在其声明的回收序列中是否被正确揭示
- 回收时机是否恰当（对应当前叙事节奏的合理时间点）
- 回收方式是否有冲击力（不应草率带过）

### 3. 无悬浮伏笔
- 是否有 `foreshadowing.md` 中声明的伏笔在实际三层文件中完全找不到对应实现（plant 和 payoff 都缺失）
- 是否有三层文件中出现的不明信息暗示了未被规划的伏笔（应建议补充到 foreshadowing.md 中）

## 输出格式

```
# 伏笔回收检查报告

## 1. 铺设完整性
- [F-02] 声明铺设于 S1-3，在 scenes/S1-3.md 与 beats/S1-3.md 中已找到 2 处相关铺设 → ✅
- [F-05] 声明铺设于 S2-1，未在任何文件找到对应场景 → ❌ 建议在 S2-1 的场景层中补入该信息的初次披露

## 2. 回收完整性
- [F-01] 声明回收于 S1-5，实际在 beats/S1-5.md: B-SC-S1-5-02-3 中回收 → ✅（但回收方式较平淡，建议加强戏剧张力）
- [F-03] 声明回收于 S3-2，但当前 S3-2 尚未生成，属于合理未完成状态

## 3. 悬浮伏笔
- 无

## 汇总
- 铺设缺失：N
- 回收缺失：M
- 未规划伏笔参考：K
```
