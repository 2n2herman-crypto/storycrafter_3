# 前端架构设计（v4）

> **交互模型**：纯对话交互 + 资产卡片展示，所有创作过程通过聊天驱动

---

## 一、组件树

```
App
├── HeaderBar                    ← 顶部标题栏
├── DiffLayout (三栏可拖拽布局)
│   ├── [左] BaselinePanel       ← 选中卡片的上一版本内容（基线）
│   ├── [中] CurrentPanel        ← 选中卡片的当前内容 + Diff 对比
│   └── [右] AssetCardPanel      ← 按 group 分组的资产卡片列表
│       └── AssetCard[]          ← 单个卡片（状态显示 + 点击选择）
└── BottomPanel
    ├── ChatInput                ← 用户输入框（Enter 发送）
    ├── FileImporter             ← 文件导入（参考素材/资产卡片）
    └── ChatHistory              ← 对话历史列表
```

---

## 二、交互流程

```
1. 用户打开应用
   ├── 左侧：基线面板（空）
   ├── 中间：当前内容面板（空）
   ├── 右侧：资产卡片列表（全部 pending）
   └── 底部：聊天输入框

2. 用户输入："我想写一个关于太空歌剧的故事"
   ├── ChatInput → sendMessage
   ├── Orchestrator 处理（Function Calling 调度）
   ├── Tool 执行 → 生成 worldbuilding.md
   ├── 前端自动刷新
   └── 系统回复："已生成世界观设定，你可以在右侧看到..."

3. 用户输入："再帮我生成几个角色"
   ├── Orchestrator 调用 generate_characters Tool
   ├── 前端自动刷新
   └── 系统回复："已生成角色设定..."

4. 用户输入："完善一下第二幕的冲突"
   ├── Orchestrator 调用 refine_act_map Tool
   ├── 前端自动刷新
   └── 系统回复："已完善第二幕的冲突升级..."

5. 用户点击右侧资产卡片
   ├── 左侧显示该文件的上一版本内容
   ├── 中间显示当前内容 + Diff 高亮
   └── 用户可以直观看到修改前后差异
```

全程不需要点击任何确认按钮——所有创作过程通过对话驱动。

---

## 三、组件说明

### BottomPanel

底部面板只包含三项：

```
BottomPanel
├── ChatInput     ← 文本输入框，Enter 发送消息
├── FileImporter  ← 拖拽或选择 .md 文件导入
└── ChatHistory   ← 显示用户和系统的对话记录
```

### AssetCardPanel

右侧资产卡片面板，按 group 分组展示：

| Group 分组 | 包含文件 |
|-----------|---------|
| **基础设定** | worldbuilding.md, characters.md, plot_synopsis.md |
| **大纲结构** | act_map.md, sequence_list.md |
| **微观精铸** | scene_beat_outline.md |

所有文件始终可见，不按阶段过滤。未生成的文件显示为 `pending` 状态。

### AssetCard

资产卡片的状态简化为三种：

```
pending   → ⚪ 待生成（灰色）
generated → 🔵 已生成（蓝色）
modified  → 🟡 已修改（黄色）
```

- 文件生成后进入 `generated`
- 内容被 Tool 修改后进入 `modified`
- 不再有 `approved`（已确认）和 `locked`（已锁定）状态

### FileImporter

文件导入组件，支持拖拽或点击选择 `.md` 文件。提供两种导入模式：

**模式 1：参考素材导入（MVP）**
```
用户拖入 reference.md
  → FileImporter 读取文件内容（< 100KB）
  → 以系统消息形式追加到 ChatHistory：
    "📎 参考素材 [filename]：（前 200 字预览...）"
  → 后续对话中 Orchestrator 和 Tool 可在 user_revision_instruction 中引用此素材
  → 不写入资产文件，不影响文件状态
```

**模式 2：覆盖资产导入（进阶）**
```
用户拖入完整的 .md 文件，选择覆盖某个已有资产
  → 选择目标资产文件（如 worldbuilding.md）
  → 通过 fileManager.writeFile 直接覆盖写入
  → 该资产卡片状态变为 modified
  → refreshAllFiles() 刷新 UI
```

**交互流程**：拖入文件后弹出选择器——"导入为参考素材" 或 "覆盖资产卡片"。MVP 只实现参考素材模式。

### BaselinePanel

左侧基线面板显示选中卡片的**上一版本内容**：

- 如果文件处于 `modified` 状态，显示修改前的内容作为对比基线
- 如果文件处于 `generated` 状态，只显示当前内容（无基线对比）
- 未选中卡片时显示空状态提示

### CurrentPanel

中间当前面板显示选中卡片的**当前内容**：

- 当 `baselineContent` 和 `content` 都存在且不同时，启用 Diff 视图
- 使用 `diffWords` 进行词级对比，`<ins>` / `<del>` 标签高亮
- 当只有一个版本时，使用普通 `<pre>` 渲染

## 四、Store 设计

### AssetStore

```typescript
interface AssetState {
  content: string           // 当前文件内容
  status: AssetStatus       // pending | generated | modified
}

interface AssetStore {
  assets: Record<string, AssetState>

  init: (fm: FileManager) => Promise<void>
  selectCard: (path: string) => Promise<void>
  updateContent: (path: string, content: string) => void
  refreshFile: (path: string) => Promise<void>
  refreshAllFiles: () => Promise<void>
  getAssetList: () => AssetCardData[]
  clearAll: () => void
}
```

### ChatStore

```typescript
interface ChatStore {
  messages: ChatMessage[]
  isProcessing: boolean

  init: (dispatcher: ToolDispatcher) => void
  sendMessage: (content: string) => Promise<void>
  addMessage: (msg: ChatMessage) => void
  clearMessages: () => void
  setProcessing: (v: boolean) => void
}
```

- `sendMessage()` 直接调用 Orchestrator 的调度循环
- 不需要单独的 IntentPurifier（功能被 Function Calling 吸收）

### PhaseStore

v4 不再使用阶段管理。前端没有 `PhaseSelector` 组件，所有交互通过 Chat 驱动。

### UIStore

```typescript
interface UIStore {
  selectedCard: string | null    // 当前选中的资产卡片
  isProcessing: boolean          // 是否正在处理
  panelWidths: { left, center, right }  // 三栏宽度
}
```

## 五、数据流

```
用户输入
  │
  ▼
ChatStore.sendMessage()
  │
  ├── 防抖检查（isProcessing中则丢弃）
  ├── 添加用户消息到 ChatHistory
  ├── 设置 isProcessing = true
  │
  ▼
Orchestrator.processUserInput(userInput)
  │
  ├── ① 计算可用 Tool（from ToolRegistry）
  ├── ② 进入调度循环（最多 5 轮）
  │     ├── LLM Function Calling → tool 选择
  │     ├── 执行 Tool → FileManager.write
  │     └── 循环/结束
  ├── ③ 返回响应文本
  │
  ▼
ChatStore 处理结果
  ├── 刷新 AssetStore（refreshAllFiles）
  ├── 添加系统回复到 ChatHistory
  ├── 设置 isProcessing = false
  │
  ▼
UI 自动刷新
  ├── AssetCardPanel 更新卡片列表
  ├── CurrentPanel 更新当前内容
  └── ChatHistory 显示新消息
```
