# 01 · 产品功能更新概要（v6.6）

> 面向产品/使用者视角，说明 v6.6 给 StoryCrafter 带来的能力变化。技术实现见 [03_产品设计文档.md](03_产品设计文档.md) 与 [04_开发方案.md](04_开发方案.md)。

---

## 一、v6.6 一句话

StoryCrafter 从"只会写一种剧本"升级为**四产品创作平台**——你先选定要创作**小说 / 剧本 / 长剧脚本 / 短剧脚本**中的哪一个，系统据此自动配置整条创作管线的结构规格与写作风格，并支持你**用任意形态的文件作为创作起点**。

---

## 二、四大产品方向

创作开始时选定一个产品方向，全程锁定（不可中途切换，切换需重置）。

### 2.1 小说
- **形态**：卷/部 → 章 → 场景 → 段落。
- **成文特色**：允许心理描写、叙述者声音、人称视角控制，节奏可张可弛。
- **产出**：章节化的小说正文。

### 2.2 剧本（电影/单本剧）
- **形态**：经典三幕 → 序列 → 场 → 节拍。
- **成文特色**：视听化表达，对白驱动，禁大段内心独白，标准剧本格式。
- **产出**：场次化的剧本正文。

### 2.3 长剧脚本（连续剧）
- **形态**：叙事大阶段 → **集（一序列=一集）** → 场景 → 节拍。
- **成文特色**：单集完整弧线 + 集间钩子衔接，多线并行，视听化。
- **产出**：一集一个 `chapters/S{幕}-{序}.md`。

### 2.4 短剧脚本（竖屏微短剧）
- **形态**：全剧大阶段 → **多集弧（每幕恰含 1 序列，8-15 集）** → 集（一集一场景）→ 4 拍微循环。
- **成文特色**：脉冲式叙事、钩子/摩擦/尖峰/钉四拍、每集反转、集间镜头分解注释。
- **产出**：一个序列(=一幕=多集)打包进单个 `chapters/S{幕}-{序}.md`。

> **关键**：四产品**共用同一套文件结构**（`sequences/*.md` + `chapters/*.md`），差异全部体现在**每层的内容语义与数量规格**上——这是产品的结构性特色，也是让同一引擎支撑四种叙事体量的根基。

## 三、新能力：任意文件输入范式

v6.6 之前只能从"一句话概念"起步。v6.6 新增**输入归一化**，让用户可以用多种起点开始创作：

| 输入形态 | 示例 | 归一化去向 |
|----------|------|-----------|
| **概念文本** | "一个程序员穿越成皇帝" | → user_requirements 种子 |
| **梗概/大纲文件** | 已写好的故事梗概 .md/.txt | → act_map / sequence_list 种子 |
| **已有原文** | 一部已写了几章的小说 | → 反向抽取 worldbuilding/characters + chapters 存量 |
| **参考资料包** | 世界观设定集、人物小传 | → worldbuilding / characters 种子 |
| **结构化大纲** | 分幕分集表格 | → act_map / sequence_list 种子 |

**用户可感知的行为**：
- 上传/粘贴任意文件后，系统先跑一次"输入归一化"，把它拆解进对应的设计资产卡片，用户可在此基础上继续创作或修订，而非从零开始。
- 归一化是**建议性**的——生成的是可编辑的种子资产，不覆盖用户后续手动调整。

---

## 四、用户视角的创作流程

```
① 选产品方向（小说/剧本/长剧/短剧）        —— 会话级锁定
② （可选）投喂输入文件 → 输入归一化生成种子资产
③ 设计期：世界观 → 角色 → 幕 → 序列 → 伏笔 → 支线 → 场景节拍
      （各层的数量规格、命名语义已按所选产品自动配置）
④ 锁定设计（Phase Gate）
⑤ 成文期：调用该产品专属的写作 Agent 逐单元产出正文
      （小说写小说、短剧写短剧，风格不串味）
⑥ 一致性检查闭环（伏笔回收、角色不 OOC、按产品标准审查）
```

---

## 五、相对旧版本，用户能感知的变化

| 维度 | v6.4 及以前 | v6.6 |
|------|------------|------|
| 支持产品 | 隐式单一（类中长剧） | 小说 / 剧本 / 长剧 / 短剧 四选一 |
| 起点 | 仅一句话概念 | 任意文件输入 + 归一化 |
| 结构规格 | 写死（3-6 场景/12-30 拍/2000-4000 字） | 按产品自动配置，规格随产品变 |
| 成文风格 | 一套通吃（视听化+禁心理） | 按产品特化，小说可心理描写 |
| 短剧支持 | 无 | 4 拍微循环 + 镜头分解 + 每幕1序列约束 |
| 结构稳定性 | 靠 prompt 自觉 | 参数由产品档案注入 + validator 按档案校验 |

## 六、不在 v6.6 承诺范围内（避免预期错位）

- **中途切换产品**：不支持，需重置重来。
- **同一作品混合产品**：一个作品一个产品方向。
- **并行批量成文的强一致性**：v6.5 设想的并行多章仍有角色/伏笔一致性风险，v6.6 以**稳定串行**为默认，并行作为后续可选优化（见 04 风险节）。
- **输入归一化的完美还原**：从已有原文反向抽取设定是"尽力而为"的建议稿，不保证 100% 覆盖。
