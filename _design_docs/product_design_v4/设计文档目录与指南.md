# StoryCrafter 3 v4 设计文档目录与指南

> **版本**：v4（Tool 调度架构）
> **状态**：设计阶段，待实现

---

## 核心原则

1. **阅读优先** — 每次实现前，先读完对应的设计文档再写一行代码
2. **设计文档唯一依据** — 架构决策以本文档体系为准，不自行发明
3. **不变铁律** — Orchestrator 绝不自己编故事内容、Agent 输出必须是纯 MD + `<<<TAG>>>` 包裹、校验规则动态读取

## 阅读顺序建议

1. **第一遍（概览）**：读 #1 `STORYCRAFTER_V4.MD` → 了解 v4 的核心理念
2. **第二遍（核心架构）**：读 #2 `tool系统架构设计.md` → #3 `上下文隔离与剪裁协议.md` → #4 `orchestrator调度引擎设计.md`
3. **第三遍（周边）**：读 #5 `前端架构设计.md` → #6 `agent注册与扩展指南.md`
4. **第四遍（技术参考）**：读 #9 `技术开发文档.md` → 了解完整模块设计和编码细节
5. **编码参考**：对照 #7 `数据接口契约文档_v4.md` + #8 `subagent提示词全集.md`
6. **开发前必读**：读 #10 `开发规划.md` → 确认当前在哪个 Step，需要操作哪些文件
