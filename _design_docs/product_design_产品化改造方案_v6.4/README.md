## 背景

v6.1 建立了「设计期 ↔ 写作期」两阶段闸门，script_writer subagent 仅在设计资产锁定后才进入 FC 面；v6.2/v6.3 完成了 scene_beats 纵切流水线、checker 切面对齐与前端聚合卡。

当前 script_writer 的能力是**基础铺展**——将单条 `sequences/<ID>.md` 的表格化场记展开为带对白描写的剧本正文。v6.4 要在此基座上注入三项专业写作能力：序列→章节的跨切片整理与伏笔感知组织、角色言行一致性与动态塑造、视听化转译。同时前端需配合锁定机制提供更适合写作期的资产浏览体验。
