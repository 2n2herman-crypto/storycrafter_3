## 一、ToolSpec

```typescript
/**
 * Tool 身份卡
 * 每个 Tool 是一个 Subagent 的 Function Calling 封装
 */
interface ToolSpec {
  /** 唯一标识符，作为 Function Calling 的 function.name */
  id: string

  /** 人类可读名称（前端展示用） */
  name: string

  /**
   * Tool 描述（最关键字段）
   * 直接作为 function.description 传给 LLM
   * 必须让 LLM 能准确判断何时调用此 Tool
   */
  description: string

  /** Subagent System Prompt 文件路径 */
  systemPromptFile: string

  /** 上下文隔离边界——只读取这些文件 */
  reads: string[]

  /** Tool 执行后写入的文件 */
  writes: string[]

  /** 输出校验 TAG（<<<TAG_START>>> / <<<TAG_END>>>） */
  outputTags: string[]

  /** 前端分组展示标签 */
  group: string

  /**
   * 可选：依赖文件列表
   * 所有文件都存在时 Tool 才可用
   */
  dependsOn?: string[]
}
```

### buildFunctionSpec

```typescript
/**
 * 从 ToolSpec 构建 Function Calling 参数
 */
function buildFunctionSpec(tool: ToolSpec): object {
  return {
    type: 'function',
    function: {
      name: tool.id,
      description: tool.description,
      parameters: {
        type: 'object',
        properties: {
          instruction: {
            type: 'string',
            description: `传递给 ${tool.name} 的具体修改指令`
          }
        },
        required: ['instruction']
      }
    }
  }
}
```

## 四、AssetCardData

```typescript
interface AssetCardData {
  path: string
  filename: string
  group: string
  status: AssetStatus
}

type AssetStatus = 'pending' | 'generated' | 'modified'
```

### 状态转换规则

```
pending  → generated  : 文件首次被 Tool 生成（writeFile 创建）
generated → modified  : 文件已被 Tool 写入过，再次被不同 Tool 或同一 Tool 的新调用写入
pending  → (无直接跳转) : 必须先经过 generated
```

- **pending** — 文件从未被写入，初始状态
- **generated** — 文件至少被写入过一次
- **modified** — 文件在 `generated` 状态下被再次写入时自动切换

**判断逻辑**：AssetStore 在 `refreshAllFiles()` / `refreshFile()` 中检测：文件存在且有内容 → 如果之前状态是 `generated`，新内容与缓存不一致则标记 `modified`；如果之前是 `pending`，则标记 `generated`。

## 六、数据流

```
用户输入
  │
  ▼
ChatStore.sendMessage()
  │
  ├── 防抖检查（isProcessing中则丢弃）
  ├── 添加用户消息到 ChatHistory
  ├── 设置 isProcessing = true
  │
  ▼
Orchestrator.processUserInput(userInput)
  │
  ├── ① 计算可用 Tool（from ToolRegistry）
  ├── ② 进入调度循环（最多 5 轮）
  │     ├── LLM Function Calling
  │     ├── 执行 Tool → FileManager.write
  │     └── 循环/结束
  ├── ③ 返回响应文本
  │
  ▼
ChatStore 处理结果
  ├── 刷新 AssetStore（refreshAllFiles）
  ├── 添加系统回复到 ChatHistory
  ├── 设置 isProcessing = false
  │
  ▼
UI 自动刷新
  ├── AssetCardPanel 更新卡片列表
  ├── CurrentPanel 更新当前内容
  └── ChatHistory 显示新消息
```
