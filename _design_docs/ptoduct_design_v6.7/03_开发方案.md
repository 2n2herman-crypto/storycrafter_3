# v6.7 序列-场景-节拍链路重构 · 开发方案

> 技术设计与关键代码形态。实现顺序见 [04_开发步骤.md](./04_开发步骤.md)。
> 所有代码集中在 scene_beats 三层：SKILL.md（prompt）、structuralChecks.ts（校验）、orchestratorEngine.ts（编排）。

## 0. 现状回顾（改造起点）

- `PIPELINE_REGISTRY.scene_beats.steps = [scene_designer, beat_writer]`，`runPipeline` 顺序跑两步 LLM，`assembleSequenceOutline` 拼装落盘。
- `executeTool` 命中 `PIPELINE_REGISTRY` 时，**强制要求合法 target**，否则早退拒收。
- dispatch loop 串行执行 tool_calls。
- 节拍是整序列一张 6 列表，`checkBeatTable` 校验整表。

## 1. 目标架构

```
executeTool(scene_beats, instruction, target?)
   │
   ├─ target 为空        → runBatchPipeline(...)   ← 全序列串行批量（新增）
   └─ target 合法(归约后) → runSequencePipeline(seqId, ...)  ← 单序列三段式（精修）
                                │
   runBatchPipeline:            │
     解析 sequence_list → [seqId...]
     for (seqId of seqIds) { await runSequencePipeline(seqId) }   ← 串行循环，不并发
     汇总 → 单 ToolResult
                                ▼
   runSequencePipeline(seqId):
     ① 建档   writeFile(sequences/<seqId>.md, skeleton)      [0 LLM]
     ② 场景   scene_designer → scenesMd(内存)                 [1 LLM, checkSceneTable]
     ③ 节拍   parseSceneIds(scenesMd) → [SC...]
              for SC in [SC...]:  beat_writer(<target_scene=SC>) → blockMd  [1 LLM/场景, checkBeatBlocks]
     收口     assembleSequenceDoc(seqId, scenesMd, Map<SC, blockMd>) → 覆写落盘
```

> 偏差 ③（并行）本次不做：`runBatchPipeline` 用 `for...await` 串行循环，不引入并发池。未来扩展为并行时，在此处套 `runWithConcurrency` + `PIPELINE_CONCURRENCY` 即可。

### 3.3 新增：块解析 + 单场景节拍校验

```ts
/** 节拍块必需字段 */
const BEAT_BLOCK_FIELDS = ['类型', '动作', '情绪', '伏笔'] as const

interface BeatBlock { id: string; fields: Record<string, string> }

/**
 * 解析节拍块序列。块头 `[BEAT B-...]` 整行锚定，字段行 `字段: 值` 非贪婪首冒号。
 * 块间空行/`<!-- -->` 注释容忍；块内无法识别的非空行计入 strayLines 供报错。
 */
export function parseBeatBlocks(md: string): { blocks: BeatBlock[]; strayLines: string[] } {
  const lines = md.split(/\r?\n/)
  const blocks: BeatBlock[] = []
  const strayLines: string[] = []
  let cur: BeatBlock | null = null
  const headRe = /^\[BEAT\s+(B-SC-[A-Z]\d+-\d+-\d{2}-\d+)\]\s*$/
  const fieldRe = /^(\S+?):\s*(.*)$/
  for (const raw of lines) {
    const line = raw.trim()
    if (!line || line.startsWith('<!--')) continue
    const h = headRe.exec(line)
    if (h) { cur = { id: h[1], fields: {} }; blocks.push(cur); continue }
    const f = fieldRe.exec(line)
    if (f && cur) { cur.fields[f[1].trim()] = f[2].trim(); continue }
    strayLines.push(line)   // 块外/无法识别行
  }
  return { blocks, strayLines }
}

/**
 * 单场景节拍块校验。sceneId = 本次 <target_scene>，用于反查块头内嵌 SC-ID 一致性。
 * @returns null=通过；string=首条违规
 */
export function checkBeatBlocks(md: string, sceneId: string, vset: ValidationSet): string | null {
  const { blocks, strayLines } = parseBeatBlocks(md)
  if (strayLines.length > 0) {
    return `存在无法识别的非字段行（首条："${strayLines[0].slice(0, 40)}"）；每行须为 \`[BEAT ...]\` 块头或 \`字段: 值\``
  }
  if (blocks.length === 0) return '未解析出任何 [BEAT ...] 节拍块'

  let lastType: string | null = null
  for (let i = 0; i < blocks.length; i++) {
    const { id, fields } = blocks[i]
    if (!BEAT_ID_REGEX.test(id)) return `第 ${i + 1} 块 B-ID "${id}" 格式非法（应形如 B-SC-S1-1-01-1）`
    // 内嵌 SC-ID 必须 = 当前场景
    const embedded = id.replace(/^B-/, '').replace(/-\d+$/, '')
    if (embedded !== sceneId) return `第 ${i + 1} 块 "${id}" 内嵌场景 "${embedded}" ≠ 目标场景 "${sceneId}"`
    // 字段齐全
    for (const key of BEAT_BLOCK_FIELDS) {
      if (!(key in fields)) return `第 ${i + 1} 块（${id}）缺字段 "${key}:"`
    }
    // 类型词库
    const primary = vset.beatTypeVocab.find((k) => fields['类型'].startsWith(k))
    if (!primary) return `第 ${i + 1} 块类型 "${fields['类型']}" 未命中词库 {${vset.beatTypeVocab.join(',')}}`
    // 相邻不同类型（档案开关）
    if (vset.enforceAdjacentTypeRule && lastType === primary) {
      return `第 ${i + 1} 块与上一块类型均为 "${primary}"，违反"相邻不得同类型"`
    }
    lastType = primary
  }
  return null
}

/** 数块头（供软校验计数） */
export function countBeatBlocks(md: string): number {
  return (md.match(/^\[BEAT\s+B-SC-[A-Z]\d+-\d+-\d{2}-\d+\]\s*$/gm) ?? []).length
}
```

> 注意：跨表引用完整性（旧 `checkBeatTable` 第 6 项）在新架构下**天然成立**——因为节拍是引擎按场景表里的 SC-ID 逐个发起的，`<target_scene>` 就来自场景表，不存在孤儿场景。校验只需反查内嵌一致性即可。

### 4.2 新增 runBatchPipeline（全序列串行批量）

```ts
private async runBatchPipeline(
  subagent: SubagentSpec, pipe: PipeRegistryValue,
  instruction: string, history?: ConversationTurn[],
): Promise<ToolResult> {
  // ① 解析 sequence_list.md 得全部序列 ID
  const seqListMd = await safeRead(this.fileManager, 'sequence_list.md')
  const seqIds = this.parseSequenceIds(seqListMd)   // 正则扫 S{幕}-{序}，去重排序
  if (seqIds.length === 0) {
    return { success: false, error: '未能从 sequence_list.md 解析出任何序列 ID，请先生成序列清单', skillName: subagent.name }
  }

  this.emit('tool_start', { toolId: subagent.id, toolName: subagent.name,
    message: `串行批量铺设 ${seqIds.length} 个序列` })

  // ② 串行 for 循环（偏差③并行本次不做，不引入并发池）
  const results: ToolResult[] = []
  for (const seqId of seqIds) {
    const r = await this.runSequencePipeline(subagent, pipe, seqId, instruction, history)
    results.push(r)
  }

  // ③ 汇总
  const ok = results.filter((r) => r.success)
  const failed = results.filter((r) => !r.success)
  const writes = ok.flatMap((r) => r.writes ?? [])
  const warnings = [
    ...ok.flatMap((r) => r.warnings ?? []),
    ...failed.map((r, i) => `序列 ${seqIds[results.indexOf(r)]} 失败：${r.error}`),
  ]
  return {
    success: ok.length > 0,
    writes,
    output: '',
    skillId: pipe.beatStep.skillId,
    skillName: subagent.name,
    error: ok.length === 0 ? '全部序列生成失败' : undefined,
    warnings: warnings.length > 0 ? warnings : undefined,
  }
}
```

> 不新增 `runWithConcurrency` 并发池小工具——偏差 ③（并行）本次不做。未来扩展并行时再加 `runWithConcurrency<T,R>(items, limit, worker)` 套在 ② 处即可，汇总逻辑 ③ 不变。

### 4.3 改造 runPipeline → runSequencePipeline（单序列三段式）

```ts
private async runSequencePipeline(
  subagent: SubagentSpec, pipe: PipeRegistryValue, seqId: string,
  instruction: string, history?: ConversationTurn[],
): Promise<ToolResult> {
  const subSkills = getSkills(subagent.id)
  const sceneSkill = subSkills.find((s) => s.skillId === pipe.sceneStep.skillId)!
  const beatSkill  = subSkills.find((s) => s.skillId === pipe.beatStep.skillId)!
  const vset: ValidationSet = this.profileLock?.validation ?? DEFAULT_VALIDATION_SET
  const finalPath = `sequences/${seqId}.md`
  const priorDraft = await safeRead(this.fileManager, finalPath)   // REFINE baseline

  // ① 建档（零 LLM）：先落骨架占位，让 UI 立刻出现卡片
  await this.fileManager.writeFile(finalPath, this.buildSequenceSkeleton(seqId))

  // ② 场景表（1 次 LLM，checkSceneTable）
  const scenesMd = await this.runSingleStep(subagent, sceneSkill, seqId, instruction, history,
    { structuralCheck: (x) => checkSceneTable(x), extras: this.designExtras(priorDraft) })
  if (scenesMd == null) return this.pipelineFail(subagent, seqId, '场景表', finalPath)

  // ③ 逐场景节拍（每场景 1 次 LLM，checkBeatBlocks）
  const sceneIds = [...extractSceneIds(scenesMd)]        // structuralChecks 导出
  const beatBySc = new Map<string, string>()
  for (const sc of sceneIds) {
    const blockMd = await this.runSingleStep(subagent, beatSkill, seqId, instruction, history, {
      structuralCheck: (x) => checkBeatBlocks(x, sc, vset),
      extras: [
        { label: 'prev_scenes', content: scenesMd },
        { label: 'target_scene', content: this.sliceSceneRow(scenesMd, sc) },  // 只喂该场景那一行
        ...this.designExtras(priorDraft),
      ],
    })
    beatBySc.set(sc, blockMd ?? `<!-- 待补节拍 ${sc}（生成失败） -->`)  // 降级不阻断
  }

  // 收口：拼装覆写落盘
  const finalMd = assembleSequenceDoc(seqId, scenesMd, sceneIds, beatBySc)
  await this.fileManager.writeFile(finalPath, finalMd)

  const softWarnings = this.runPipelineSoftValidation(scenesMd, beatBySc)
  return { success: true, writes: [finalPath], output: '',
    skillId: beatSkill.skillId, skillName: beatSkill.name,
    warnings: softWarnings.length > 0 ? softWarnings : undefined }
}
```

`runSingleStep` 把原 runPipeline 内「读 reads → assembleContext → appendExtraLabels → sendMessage ×retry → validateOutput」的重试块抽成私有方法（返回 extracted 内容或 null），供段②③复用，避免复制粘贴。`designExtras(priorDraft)` 统一拼 `<current_target>` baseline + `<product_profile>`。

### 4.4 收口 assembleSequenceOutline → assembleSequenceDoc

```ts
function assembleSequenceDoc(
  seqId: string, scenesMd: string, sceneIds: string[], beatBySc: Map<string, string>,
): string {
  const scenesTable = scenesMd.trim()   // 场景仍是表格，直接 trim（删 extractTableBlock）
  const beatSections = sceneIds.map((sc) =>
    `#### ${sc}\n\n${(beatBySc.get(sc) ?? '').trim()}`).join('\n\n')
  return [
    '<<<SCENE_BEAT_OUTLINE_START>>>',
    `# ${seqId}`, '',
    '### 场景表', '', scenesTable, '',
    '### 节拍', '', beatSections,
    '<<<SCENE_BEAT_OUTLINE_END>>>',
  ].join('\n')
}
```

外层 TAG + `# <seqId>` 标题保留 → 下游 `<current_sequence_beats>` 契约不破。

### 4.5 辅助函数

```ts
private parseSequenceIds(seqListMd: string): string[] {
  const set = new Set<string>()
  const re = /\bS\d+-\d+\b/g   // 扫序列 ID；表格首列/正文均命中
  let m: RegExpExecArray | null
  while ((m = re.exec(seqListMd)) !== null) set.add(m[0])
  return [...set].sort()
}

private buildSequenceSkeleton(seqId: string): string {
  return `<<<SCENE_BEAT_OUTLINE_START>>>\n# ${seqId}\n\n### 场景表\n\n*（生成中…）*\n\n### 节拍\n\n*（生成中…）*\n<<<SCENE_BEAT_OUTLINE_END>>>`
}

/** 从场景表抽出 sc 那一行原样（供 <target_scene> 精准注入，模型只看这一个场景） */
private sliceSceneRow(scenesMd: string, sc: string): string {
  const line = scenesMd.split(/\r?\n/).find((l) => l.includes(sc))
  return line ? line.trim() : sc
}
```

### 4.6 软校验改块计数

`runPipelineSoftValidation(scenesMd, beatBySc)`：场景数仍数表格行（`countTableDataRows(scenesMd)`），节拍数改 `[...beatBySc.values()].reduce((n, md) => n + countBeatBlocks(md), 0)`，平均值判区间逻辑不变。删除对整表 beatsMd 的 `countTableDataRows`。

## 7. 验证方法

1. `npx tsc -b` 通过（删函数无残留引用）。
2. `npm run dev` → 选短剧 → 建好 act_map/sequence_list → 说「铺所有序列的场景节拍」：
   - 观察资产面板逐张出现 sequences 卡片（建档骨架先现，再逐步填充；串行批量不并发）；
   - 每个文件：场景表（表格）+ 节拍（`#### SC-ID` 分组的块）；
   - **重点验偏差①**：每个场景下都有节拍块，无跳过；
   - **重点验偏差②**：beat_writer 单场景调用几乎不 retry。
3. 精修：「重写 S1-2 场景节拍」→ 只 S1-2 变，其余不动；旧表格文件能转块格式。
4. 进写作期调 writer：`<current_sequence_beats>` 注入新格式，正常产出。
5. 软校验 warning 仍能报场景/节拍数越界。
