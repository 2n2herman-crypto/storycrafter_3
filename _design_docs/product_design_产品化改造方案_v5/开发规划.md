# V5 开发规划

> v4 → v5 的代码改造步骤。按顺序执行，每一步说明改什么文件、怎么改、改完怎么验证。

---

## 总览

v4 有 **14 个 Tool**（6 个生成 + 6 个精炼 + reset + orchestrator），gen/refine 分离，dependsOn 硬闸门过滤。

v5 改造为 **9 个 Tool**（gen/refine 合并为 5 个 + 3 个新增 + reset），**dependsOn 全部放行**，新增 **审计闭环**。

---

## Step 1 — types/index.ts：加审计字段

**改什么**：`SchedulerState` 接口加 2 个字段。

```typescript
export interface SchedulerState {
  currentRound: number
  maxRounds: number
  toolsCalled: string[]
  toolResults: ToolResult[]
  auditRound: number       // ← 新增，当前审计轮次
  maxAuditRounds: number   // ← 新增，固定 3
}
```

**验证**：`npx tsc -b` 通过。编译不报错就算过。

```typescript
// 改前（v4）：
export function getAvailableTools(existingFiles: Set<string>): ToolSpec[] {
  return TOOL_REGISTRY.filter((tool) => {
    if (!tool.dependsOn || tool.dependsOn.length === 0) return true
    return tool.dependsOn.every((f) => existingFiles.has(f))
  })
}

// 改后（v5）：
export function getAvailableTools(): ToolSpec[] {
  return TOOL_REGISTRY
}
```

```typescript
export const HIDDEN_ASSET_PATHS = ['_check_report.md', 'draft_history.md']
```

## Step 4 — assetStore.ts：隐藏文件过滤

**改什么**：`getAssetList()` 方法加一行 `.filter()`。

```typescript
// 改前：
getAssetList: () => {
  const { assets } = get()
  return Object.entries(assets).map(([path, state]) => { ... })
}

// 改后：
getAssetList: () => {
  const { assets } = get()
  return Object.entries(assets)
    .filter(([path]) => !path.startsWith('_'))  // ← 只加这行
    .map(([path, state]) => { ... })
}
```

`_check_report.md` 和 `draft_history.md` 都是下划线开头，自动被过滤掉，不出现在 AssetCard 列表里。

**验证**：`npx tsc -b` 通过。

## 关于并行执行（可选）

Step 5 中还有一个可选优化：把 Tool 串行执行改成并行。

位置在第 5 步的 5.6 之前，即找到 `for (const toolCall of callsToProcess)` 那个循环（约第 300 行）：

```typescript
// 改前：串行执行
for (const toolCall of callsToProcess) {
  const result = await this.executeTool(toolSpec, instruction)
  // ... 结果处理（5.6 的内容）
}

// 改后：并行执行
const results = await Promise.all(
  callsToProcess.map(async (toolCall) => {
    const toolId = toolCall.function.name
    const toolSpec = getTool(toolId)
    if (!toolSpec) return { toolCall, result: { success: false, error: `未知工具: ${toolId}` }, toolSpec: null }
    const result = await this.executeTool(toolSpec, instruction)
    return { toolCall, result, toolSpec }
  })
)

// 统一处理 results（保持 tool_calls 的原始顺序）
for (const { toolCall, result, toolSpec } of results) {
  // ... 结果推送消息逻辑（5.6 的条件分支放在这里）
}
```

**安全前提**：v5 已合并 gen/refine，每个文件的写入者唯一，不会出现两个工具同时写同一文件的冲突。放到 Step 5 最后做，不影响前面改造。
