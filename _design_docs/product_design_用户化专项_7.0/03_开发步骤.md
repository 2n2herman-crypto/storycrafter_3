# 03 开发步骤 — 7.0 用户化专项

## 实施顺序总览

```
M0 目录准备 → M1 LLM 代理 → M2 目录调整 → M3 资产持久化+Word
→ M4 多项目+对话 → M5 Skill 运行时 → M6 文件关系预留+打磨
→ v7.2 前端美化
```

每步可独立验收，允许停在任意里程碑。每步结束跑 `npx tsc -b` 全绿。

---

**关键文件**：`server/package.json`、`server/tsconfig.json`、`server/src/index.ts`、`package.json`、`vite.config.ts`、`.gitignore`

**验收**：

## M1 LLM 代理（1–2 天）

**做什么**：
1. 后端 `server/src/services/configStore.ts`：读写 `server/data/config.json`，首启动从 `VITE_DEEPSEEK_API_KEY` env 种入，`fs.chmod 0600`
2. 后端 `server/src/services/llmProxy.ts`：读 active profile，加 `Authorization: Bearer`，POST 到 `{baseURL}/v1/chat/completions`，透传 tools/tool_choice
3. 后端 `server/src/routes/llm.ts`：`POST /api/llm/chat`（non-streaming 先行）+ `POST /api/llm/test`
4. 后端 `server/src/routes/config.ts`：`GET/PUT /api/config`（apiKey mask 返回）

6. 前端 `LLMClient` 改打 `/api/llm/chat`，摘 `dangerouslyAllowBrowser`，对外签名不变
7. 新建 `web/src/pages/Settings/`：单 profile 表单（baseURL/apiKey/model）+ 测试连接按钮

**关键文件**：`server/src/services/{configStore,llmProxy}.ts`、`server/src/routes/{llm,config}.ts`、`web/src/api/{llm,config}.ts`、`src/llm/client.ts`、`web/src/pages/Settings/`

**INV 守卫**：`LLMClient.sendMessage`/`sendMessageWithTools` 签名不变，`OrchestratorEngine` 零改

## M3 资产持久化 + Word 导入导出（2–3 天，核心整合）

**做什么**：
1. 后端 `server/src/util/fsQueue.ts`：按 projectId 分片串行队列（Map<projectId, Promise chain>）
2. 后端 `server/src/services/projectStore.ts`：项目 CRUD + `server/data/projects/<id>/` 目录管理 + `projects.json` 索引
3. 后端 `server/src/routes/projects.ts`：6 个项目端点（list/create/get/patch/delete/duplicate）
4. 后端 `server/src/routes/assets.ts`：5 个资产端点（list/read/write/delete-one/delete-all）
   - **path 校验放宽**：允许子目录 + `..`/绝对路径黑名单 + `path.resolve`+`startsWith(projectRoot)` 兜底
5. 后端 `server/src/routes/import.ts`：`POST /api/import/docx`（multer multipart + mammoth + turndown）
6. 后端 `server/src/routes/export.ts`：`POST /api/export/docx`（docx 库 md→docx）
7. 前端 `web/src/orchestrator/fileManager.ts`：新增 `HttpFileManager implements FileManager`，六方法逐字对齐
8. 前端 `web/src/api/{projects,assets,importExport}.ts`
9. 前端 `App.tsx:34`：DI 切换 `InMemoryFileManager`→`HttpFileManager`（暂用单项目 default）
10. 前端 `FileImporter.tsx`：`accept` 加 `.docx`，`BottomPanel.handleFileImport` 按扩展名分流
11. 前端 `CurrentPanel.tsx`：头部加「导出 MD / 导出 Word」按钮组
12. 前端 `web/src/utils/exportMd.ts`：纯前端 MD 导出（Blob+download）

**关键文件**：`server/src/{util/fsQueue,services/projectStore,routes/{projects,assets,import,export}}.ts`、`web/src/orchestrator/fileManager.ts`、`web/src/api/{projects,assets,importExport}.ts`、`web/src/App.tsx`、`web/src/components/{BottomBar/FileImporter,Layout/CurrentPanel}.tsx`、`web/src/utils/exportMd.ts`

**验收**：
- 刷新页面资产还在，`server/data/projects/<id>/assets/*.md` 看到真文件
- 用编辑器手改某 `.md` → 刷新 → 改动生效
- `.docx` 拖入 → 预览 → 落 `_input_raw.md` → input_normalizer 归一化通
- 导出 MD 下载 `.md`，导出 Word 下载 `.docx` 内容正确
- `scene_beats` 全量批量 → `sequences/S1-1.md` 等子目录路径正常读写

**INV 守卫**：`FileManager` 六方法签名逐字不变；`OrchestratorEngine` 构造签名不变；`validateOutput`/`assembleContext`/`types/index.ts` 零改；`_input_raw.md` → `input_normalizer` 链路零改

## v7.2 前端美化 3.0（2.5–3 天）

### 步骤 1：暗色主题（半天）
1. `web/src/styles/global.css` 加 `[data-theme="dark"]` 覆盖 ~25 个 `--color-*`
2. `web/src/styles/diff.css` diff 变量加暗色版
3. 新建 `web/src/hooks/useTheme.ts`（读写 localStorage、设 `document.documentElement.dataset.theme`）
4. `HeaderBar` 加 ☀/🌙 toggle

### 步骤 2：响应式（1–1.5 天）
1. 新建 `web/src/hooks/useResponsive.ts`（`matchMedia` 返回 `'wide'|'medium'|'narrow'`）
2. `MultiColumnLayout.tsx` 接受 `layoutMode` prop，按模式渲染（wide=现状 flex，medium=两栏+内 Tab，narrow=单栏+Tab）
3. `global.css` 加 `@media` 断点调整 `--header-height`/字号
4. `App.tsx` 根据 mode 传 `columns` 组合
5. **降级**：时间紧先只做 wide+narrow 两档

### 步骤 3：视觉调优（1 天）
1. `global.css` 加 spacing scale（`--space-1`..`--space-8`）+ 字体层级变量
2. 替换各 Module CSS 硬编码 padding/字号
3. 空状态：`AssetCardPanel`/`ChatHistory` 无内容时占位文案+图标
4. 加载骨架：新建 `web/src/components/Skeleton.tsx`，`CurrentPanel.isLoading` 显示
5. AssetCard hover/状态圆点微动画

### 步骤 4：v7.1 UI 入口融入

- 主题切换（HeaderBar 右侧）
- Settings（齿轮→独立视图）
