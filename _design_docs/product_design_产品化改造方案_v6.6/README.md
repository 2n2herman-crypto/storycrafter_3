# StoryCrafter v6.6 设计文档 · 四产品统一框架

> **一句话**：把"隐式单一中长剧"的创作管线，升级为**四产品（小说 / 剧本 / 长剧脚本 / 短剧脚本）统一框架**——四层骨架（幕-序-场-节）共享，每层语义与量级由**产品档案（ProductProfile）**声明式驱动；成文层按产品特化 Subagent；新增**输入归一化范式**支持任意文件输入。

---

## 为什么是 v6.6（相对 v6.4 / v6.4.1 / v6.5 的定位）

| 版本 | 性质 | 结论 |
|------|------|------|
| v5.3 / v6.1 | **已落地代码基线** | 四层框架 + scene_beats 流水线 + script_writer + Phase Gate |
| v6.4 / v6.4.1 / v6.5 | **未开发的 PRD 草案** | 引用的 modeStore/ModeParameters/variant/并行批处理**均未实现**；且路线互斥、边界未定义 |
| **v6.6（本文）** | **收敛后的正式设计** | 废弃 v6.4.x/v6.5 的双模式与 variant 路线，改为**产品档案驱动的统一框架** |

v6.6 **吸收** v6.4.1 的短剧叙事研究成果与 v6.5 的"直出/加速"诉求，但**重构其落地形态**：
- 不再是"短剧 vs 中长剧"两模式，而是**四产品**。
- 不再靠 variant 热插拔或纯参数注入，而是**产品档案（上游参数化 + 语义槽）+ 成文层特化 Subagent** 的分层混合路线。
- 新增此前从未涉及的**输入范式**：任意文件输入 → 归一化种子资产。

## 核心架构一图流

```
┌──────────────────────────────────────────────────────────────┐
│                     用户输入（多形态）                          │
│   概念文本 / 梗概文件 / 已有小说原文 / 参考资料包 / 结构化大纲   │
└───────────────────────────┬──────────────────────────────────┘
                            │  + 选定产品(novel/screenplay/long_drama/short_drama)
                            ▼
              ┌───────────────────────────┐
              │   ① 输入归一化区            │
              │   input_normalizer         │  任意文件 → 归一化种子资产
              └─────────────┬─────────────┘
                            ▼
              ┌───────────────────────────┐
              │   ProductProfile（产品档案）│  ← 会话级锁定，驱动下方所有层
              │   · 四层语义槽              │
              │   · 四层量级参数            │
              │   · 叙事单位映射            │
              │   · validator 校验集合      │
              │   · 绑定的成文 Subagent     │
              └─────────────┬─────────────┘
                            ▼
      ┌──────────────── ② 设计区（骨架共享，档案参数化）─────────────┐
      │ worldbuilding → characters → act_map → sequence_list          │
      │              → foreshadowing → subplots → scene_beats          │
      └───────────────────────────┬──────────────────────────────────┘
                            ▼  (Phase Gate lock)
      ┌──────────────── ③ 成文区（按产品特化，互斥可见）────────────┐
      │  novel_writer │ screenplay_writer │ long_drama_writer │        │
      │                                    │ short_drama_writer         │
      │  ↑ 仅暴露与 ProductProfile 绑定的那一个                        │
      └───────────────────────────┬──────────────────────────────────┘
                            ▼
              ┌───────────────────────────┐
              │   记忆运行时（跨调用状态）  │
              │   · behaviorTrack 角色追踪  │
              │   · foreshadowingState 伏笔 │
              │   · profileLock / 批次进度  │
              └───────────────────────────┘
```

## 四产品速览

| 产品 | 幕语义 | 序列语义 | 场景语义 | 集映射 | 成文 Subagent | 成文风格 |
|------|--------|---------|---------|--------|--------------|---------|
| **小说** | 卷/部 | 章 | 场景/段落 | 无 | `novel_writer` | 允许心理描写、叙述声音、人称 |
| **剧本** | 幕(经典三幕) | 序列 | 场 | 无 | `screenplay_writer` | 视听化、禁大段内心独白 |
| **长剧脚本** | 叙事大阶段 | **集**（1序列=1集） | 场景 | 一序列=一集 | `long_drama_writer` | 视听化、脉络展开 |
| **短剧脚本** | 全剧大阶段 | **多集弧**（每幕恰1序列） | 集（一集一场景） | 一序列=多集 | `short_drama_writer` | 脉冲式、4拍微循环、镜头分解 |

> 详细的量级参数（各层数量区间、节拍词库、伏笔寿命、成文粒度）见 [03_产品设计文档.md](03_产品设计文档.md) 的产品档案契约表。

## 三条设计红线（贯穿全套文档）

1. **骨架不变，语义可变**：`sequences/S{幕}-{序}.md` / `chapters/S{幕}-{序}.md` 文件结构在四产品间完全一致；变化的是各层语义与量级，由产品档案承载，不新增 per-episode 文件。
2. **成文层单一职责**：一个产品一个 writer，prompt 只讲该产品的写法，杜绝"视听化 vs 心理描写"这类互斥指令共存 → **输出最稳定**。
3. **参数由档案给、不靠 LLM 猜**：幕数/序列数/场景数/节拍数/词库/字数区间全部来自 ProductProfile 注入，validator 按档案校验 → 结构确定性最大化。
